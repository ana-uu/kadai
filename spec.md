# ランキングサイト仕様書

## 概要

app5.jsを参考に、3つの異なるランキング情報を管理するWebサイトを開発する。

## 開発対象サイト

### 1. FANZA売れ行きランキング
- **目的**: FANZA商品の売れ行きランキングを管理
- **独立サイト**: 単独ポートで動作
- **データ構造**:
  ```javascript
  {
    id: Number,
    rank: Number,
    title: String,
    genre: String,
    price: Number,
    releaseDate: String
  }
  ```

### 2. Amazon売れ行きランキング
- **目的**: Amazon商品の売れ行きランキングを管理
- **独立サイト**: 単独ポートで動作
- **データ構造**:
  ```javascript
  {
    id: Number,
    rank: Number,
    productName: String,
    category: String,
    price: Number,
    rating: Number
  }
  ```

### 3. 日本の人気観光地ランキング
- **目的**: 日本の人気観光地ランキングを管理
- **独立サイト**: 単独ポートで動作
- **データ構造**:
  ```javascript
  {
    id: Number,
    rank: Number,
    spotName: String,
    prefecture: String,
    category: String,
    visitorsCount: Number
  }
  ```

## 共通機能仕様

### HTTPメソッドとリソース名

| 機能 | HTTPメソッド | URL | 説明 |
|------|-------------|-----|------|
| 一覧表示 | GET | `/` | ランキング一覧を表示 |
| 詳細表示 | GET | `/:id` | 指定IDの詳細情報を表示 |
| 新規登録フォーム | GET | `/create` | 新規登録フォームを表示 |
| 新規登録処理 | POST | `/create` | 新規データを登録 |

**注意**: 各サイトは独立したポートで動作するため、ルートパスは各サイトのルートから開始

### ページ構成

1. **一覧ページ**: ランキング順にデータを表形式で表示
2. **詳細ページ**: 選択したアイテムの詳細情報を表示
3. **新規登録ページ**: 新しいランキングデータを追加

### 技術仕様

- **フレームワーク**: Express.js
- **テンプレートエンジン**: EJS
- **ポート**: 各サイトは異なるポートで動作
- **データ保存**: メモリ内配列（本格運用時はDB使用）
- **独立性**: 各サイトは完全に独立して動作



## 各サイト固有仕様

### FANZA売れ行きランキング
- ランク、タイトル、ジャンル、価格、発売日を管理
- 価格は円単位で表示

### Amazon売れ行きランキング
- ランク、商品名、カテゴリ、価格、評価を管理
- 評価は5段階（1-5）

### 日本の人気観光地ランキング
- ランク、観光地名、都道府県、カテゴリ、年間訪問者数を管理
- 訪問者数は万人単位で表示
