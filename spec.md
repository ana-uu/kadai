# ランキングサイト仕様書

## 概要

app5.jsを参考に、3つの異なるランキング情報を管理するWebサイトを開発する。

## 開発対象サイト

### 1. FANZA売れ行きランキング
- **目的**: FANZA動画の売れ行きランキングを管理
- **独立サイト**: ポート8081で動作
- **データ構造**:
  ```javascript
  {
    id: Number,
    rank: Number,
    title: String,
    genre: String,
    price: Number,
    releaseDate: String,
    actress: String,
    link: String
  }
  ```

### 2. DLsite売れ行きランキング
- **目的**: DLsite作品の売れ行きランキングを管理
- **独立サイト**: ポート8082で動作
- **データ構造**:
  ```javascript
  {
    id: Number,
    rank: Number,
    productName: String,
    genre: String,
    price: Number,
    rating: Number,
    circle: String,
    link: String
  }
  ```

### 3. メロンブックス売れ行きランキング
- **目的**: メロンブックス同人誌の売れ行きランキングを管理
- **独立サイト**: ポート8083で動作
- **データ構造**:
  ```javascript
  {
    id: Number,
    rank: Number,
    bookName: String,
    author: String,
    circle: String,
    genre: String,
    price: Number,
    link: String
  }
  ```

## 共通機能仕様

### HTTPメソッドとリソース名

| 機能 | HTTPメソッド | URL | 説明 |
|------|-------------|-----|------|
| 一覧表示 | GET | `/` | ランキング一覧を表示 |
| 詳細表示 | GET | `/:id` | 指定IDの詳細情報を表示 |
| 新規登録フォーム | GET | `/create` | 新規登録フォームを表示 |
| 新規登録処理 | POST | `/create` | 新規データを登録 |
| 編集フォーム | GET | `/edit/:id` | 編集フォームを表示 |
| 編集処理 | POST | `/edit/:id` | データを更新 |
| 削除処理 | POST | `/delete/:id` | データを削除 |

**注意**: 各サイトは独立したポートで動作するため、ルートパスは各サイトのルートから開始

### ページ構成

1. **一覧ページ**: ランキング順にデータを表形式で表示
2. **詳細ページ**: 選択したアイテムの詳細情報を表示
3. **新規登録ページ**: 新しいランキングデータを追加

### 技術仕様

- **フレームワーク**: Express.js
- **テンプレートエンジン**: EJS
- **ポート**: 各サイトは異なるポートで動作
- **データ保存**: メモリ内配列（本格運用時はDB使用）
- **独立性**: 各サイトは完全に独立して動作



## 各サイト固有仕様

### FANZA売れ行きランキング
- ランク、タイトル、ジャンル、価格、発売日、女優名、リンクを管理
- 価格は円単位で表示
- ランク変更時に他のアイテムのランクを自動調整

### DLsite売れ行きランキング
- ランク、作品名、ジャンル、価格、評価、サークル名、リンクを管理
- 評価は5段階（小数点含む）
- ランク変更時に他のアイテムのランクを自動調整

### メロンブックス売れ行きランキング
- ランク、書籍名、著者、サークル名、ジャンル、価格、リンクを管理
- 価格は円単位で表示
- ランク変更時に他のアイテムのランクを自動調整
